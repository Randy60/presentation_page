Python 3.0

Python 2.x is legacy, Python 3.x is the present and future of the language -wiki.python.org

What is Python 3.0 (aka py3k) ?

  • Python 2.0s successor
    • Python 3.0 was released in December 2008
    • Hope was for implementation to take about 5 years (behind schedule)
    • Current version is 3.5.1
  • Intentionally not backwards compatible
  • Python 3.0 is mostly the same, but there were enough changes to warrant a 3.0 version, not just a 2.x update.
    • Guido: We are mostly fixing well-known annoyances and warts, and removing a lot of old cruft.

Why Python 3.0?

  • In general, a fresh slate to build-on and usher in this future of the language
  • Implemnted to fix bugs and eliminate redundancies
  • Python 3 changes unicode structure
    • In both, there are two types of strings
      • byte sequences using ASCII encoding
      • Unicode strings
    • Unicode > ASCII: more versatile and more used
      • Unicode assigns a unique integer value to each character in every language
        • Allows characters from foreign languages, emojis, and the standard Roman letters and numerals
      • Python 2: Unicode not the default, have to mark with a “u”:
      • Python 3: All strings are Unicode by default, need a “b” to make a byte sequence

Which should you use?

  • While we learned 2.7.1, it is argued that Python 3.x is easier for beginners to learn
  • Python 3.x, however, is the future of the language as they try to phase out v. 2.7.1
  • Right now, really comes down to libraries. Newer libraries with various capabilities are only Python 3 compatible while others may be Python 2 compatible and not adjusted.

2.x vs 3.x

Python 2.7

  • More widely implemented.
  • Better Documentation and support.
  • Strong community surrounding it that python 3.x does not have.
  • Performance speed is essentially equal
  • Integer Division 3/2 = 1 : 3.0/2 = 1.5
  • It is the language people know and are used too and therefore see no need to change from it.

Python 3.0

  • Not a strong backbone of users
  • Not much documentation and support
  • Print ()
  • Integer division 3/2 = 1.5
  • All text is now Unicode
  • Some small syntax changes but the two are very similar.
  • Down the road will be more useful because it can handle more processes and is more powerful but for now people see no need to change.

Other Differences

  • raising and handling exceptions
  • the next() function
  • for-loop variables and the global namespace leak
  • comparing unorderable types
  • parsing user inputs via input()
  • returning iterable objects instead of lists
  • built-in objects (dictionaries and strings) and libraries

Implementation

  • Although Python 3.0 was released in 2008, Python 2.x continued to receive updates culminating with Python 2.7 which was released 2010.
  • Many 2.x users are not eager to switch, and 2.x is still the pre-installed language on both Mac and Lunix systems.
  • Python 2.x is the language of most current Python code, so switching to 3.x would be hard in an environment the user doesn’t control.
  • People don’t see a need to switch.
